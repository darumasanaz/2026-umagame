# 2026-umagame

## HUDの整理
- 右上に「MONEY / TARGET」をまとめて表示するようにし、従来のSCORE表記は削除しました。
- レイアウト用の余白は`game.js`内の`drawHUD()`で管理しているので、表示位置を変えたい場合は`margin`を調整してください。

## グラフィックを改善するアイデア
1. **描画ロジックの分割**  
   `drawHorse()`や`drawObstacles()`内のswitchをそれぞれ専用の関数に分けると、部品ごとに試作しやすくなります。`Path2D`や`ctx.arc()`を使って顔・耳・装飾などのシルエットを描くと、馬・門松・鯛・羽根といったモチーフが判別しやすくなります。
2. **色と陰影の追加**  
   2〜3色のレイヤーを重ねて影を付けると立体感が出ます。たとえば門松なら竹部分を明るいグラデーションに、松の葉を別パーツで描き込むと「緑色の棒」から「門松」らしく見せられます。
3. **参考画像ベースのスケッチ**  
   画像検索で元となる形を確認し、主要な輪郭線を直線or円弧の組み合わせで抽象化してからCanvasに落とし込むと、簡易イラストでもモチーフが伝わりやすくなります。

## 障害物サイズを調整する際の指針
- 現在のサイズは`game.js`の`spawnObstacle()`内で`const width = canvas.width * 0.05`のように決まっています。障害物ごとに`widthFactor`と`heightFactor`を定義した設定オブジェクトを作り、そこからサイズを参照する形にすると、門松や鳥居だけを個別に小さくする実験が楽になります。
- 例えば鳥居の高さを抑えたい場合は、`heightFactor`を`0.55`から`0.35`程度に下げる、もしくは`horse.height`に対して比率で制限し、`Math.min(canvas.height * 0.4, horse.height * 2.2)`といったクリッピングを入れると良いです。
- ジャンプで超えられるかは「高さ」だけでなく「幅」と「スピード」にも依存します。門松の横幅を`0.05`→`0.035`に調整し、同時に`speed`を少し下げれば、プレイヤーの反応猶予を確保できます。

これらの方針を組み合わせることで、見た目とゲームバランスの双方を段階的に改善していけます。
